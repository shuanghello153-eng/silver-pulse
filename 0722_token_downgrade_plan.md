# 0722 Token 降级方案（免费期结束后执行）

> 制定日期：2026-07-08 ｜ 执行窗口：2026-07-22 当天
> 背景：HY3 模型两周免费期约 07-22 结束。届时自动化任务（每日资讯更新 / 标签池迭代 / 每日摘要推送）若继续用 Hy3 将产生积分消耗。本方案在不明显牺牲质量的前提下分级降级。

## 一、现状成本结构（已实测）

| 任务 | 频率 | 主要 token 消耗点 | 是否调模型 |
|------|------|------------------|-----------|
| 每日资讯更新 | 每日 01:00 | collector 抓 44 源（网络 IO）+ 评分规则兜底 + 生成 HTML | **评分=规则，不调模型**；仅"读脚本输出+写报告"耗 token |
| 标签池迭代 | 每周一 03:00 | 扫描新标签 + 更新 config + 部署 | 纯格式化，**零模型推理** |
| 每日摘要推送 | 每日 06:00 | 读 scored_latest + 生成摘要 | 纯格式化，**零模型推理** |

**关键结论**：本流水线**评分/推荐/聚类全部是规则兜底（config 权重），不调 LLM**。token 消耗主要来自"自动化框架读脚本 stdout + 模型写执行报告"。所以降级空间大。

## 二、已落地的降本动作（07-08）

1. **标签池迭代：每日 → 每周一**（省 6/7 运行）
2. **脚本静音**（T58，见下）：collector 等详细日志写 `data/run_logs/`，只给模型一行摘要，预计每日更新再降 30–50%
3. **信源修复**（T57）：修正 4 个核心源 RSS 地址，避免反复重试/超时浪费

## 三、0722 当天执行清单（按优先级）

### P0 — 必须做（保核心质量）
- [ ] **每日资讯更新**：保持 Hy3。这是唯一真正依赖"智能"的环节（采集调度、异常判断、部署决策），Hy3 深度推理价值最高，且每日仅 1 次，成本可控。
- [ ] **确认 Hy3 免费期真实结束日期**：登录控制台核对，避免提前降级或过期后未知模型。

### P1 — 建议降级（纯格式化任务，不需要深度推理）
