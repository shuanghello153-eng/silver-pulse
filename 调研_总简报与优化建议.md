## 二、Loop 工程（你点名第一优先）——结论：骨架已具备，只差"激活+校验"

**现状真相**：我们之前几轮已落地一整套 Loop 代码（validator / loop_audit / noise_spike_guard / feedback_loop / adapt_thresholds / daily_health / visual_regression / 部署门禁）。它比很多项目扎实，但本质是**"报告型半闭环"**：

- `adapt_thresholds` 默认 `ENABLE_AUTO_THRESHOLD=False` → 阈值只是建议，不真改；
- `feedback_loop` 因你收藏少、`feedback.jsonl≈0` → 实际空转（伪闭环）；
- 循环发现不会自动写进 `.workbuddy/memory/`。

**三个真缺口**：①闭环只报告不动作；②L3反馈休眠缺样本；③无自动记忆蒸馏。

**落地路线（与你的判断一致——先校验，再闭环）**：
- **P0 校验层**：死链检测 + JSON Schema 字段断言 + 评分一致性抽样（高/低分企业各抽搜信息核对）+ 前后条数骤降硬门槛。全部零模型成本。
- **P1 迭代+记录**：你拍板激活 `ENABLE_AUTO_THRESHOLD`；新增 `loop_memorize.py` 每天自动把循环结论蒸馏进记忆；多用网站收藏让 L3 不再空转。
- **P2 谨慎闭环**：任何"AI 改自己规则/权重晋升"一律留给你确认，护栏沿用 git 备份+覆盖层+pitfalls_log。**明确不碰多 agent 编排、向量记忆、RL 自训练**（属过度工程）。

> 一句话：Loop 不是从零造，是把已有的骨架"通电"。通电后，B类评分整改、C类数据源、标签重构都能在周日自动化里自我进化，**不再每步等你签字**——这正是解你"人太慢"瓶颈的钥匙。

## 三、海外一手信息源（你定的护城河）——结论：18个已验证直连，单点故障可解

**核心发现**：用与生产采集器相同的 requests 栈实测，**18 个直连 RSS 已验证可用**：
- 最对口：`Longevity Technology`（长寿/老龄化科技融资，国内无替代）、`Fierce Healthcare/Biotech`（可替 google_news 代理）
- 政策一手：美国养老协会 `LeadingAge` + `Argentum`
- 欧洲融资：`EU-Startups`、`Silicon Canals`、`HIT Consultant`、`FemTech Insider`
- 垂直：`AgeTech Collaborative`、`StartUp Health`、`Crunchbase News`、`Senior Housing News/Hospice News/Home Health Care News/The Gerontechnologist`、`PR Newswire`、`Yahoo Finance`

**最关键的架构建议**：
1. 当前 `DIRECT_RSS_FALLBACK` **故意留空** → 立刻填入上述 18 源（代码已支持 google_news→direct rss 降级）。
2. 把已在 config 但走代理的 6 个高价值垂直源**直接改 `rss` 直连**，彻底去依赖。
3. 给 google_news 加 **Bing News RSS 二级兜底**（`https://www.bing.com/news/search?q=Q&format=rss`，同构易接），Google News 全局失败时自动降级到"直连全量+Bing"，避免当天归零。
4. `when:7d` 硬编码过短，协会/政策/研究报告源会漏抓 → 拆成独立窗口（新闻7天/协会政策30天）。
5. **保留两级相关性闸门**——新增多为整站 feed，否则 Longevity/Fierce 会灌泛科技噪音。

> 这不是"要不要加源"的问题，而是"已经验证能加、且能立刻消除断粮风险"。等你拍板即落地。

## 四、竞品抄作业——结论：3项最值钱，且我们已有雏形

14 个真实核验产品（NIC MAP / Senior Housing News / Argentum / Ziegler / LeadingAge / AgeTech Collaborative / Longevity.Technology / Rock Health / PitchBook / CB Insights / Feedly AI / Techmeme / Glasp / The Gerontechnologist）中，最该抄的 3 项**都能用现有 JSON 纯前端实现**：

1. **NIC MAP 的"角色/视角分组 + KPI 仪表盘"** → 补首页全局概览（本周新增/高价值/覆盖赛道）。
2. **Techmeme 的"事件簇多源对照"** → 把同事件多家海外源聚成一张卡，正是"以海外为镜"最佳载体。**我们已有 0.82 聚类，只差呈现**——性价比最高。
3. **CB Insights 的"市场地图"** → 已有 485 家企业+12 类，升级二维赛道全景图成本极低、辨识度最高。

报告另含 14 条建议（周报通讯、赛道榜单等次优先）。
