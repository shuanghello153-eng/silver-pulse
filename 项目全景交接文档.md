# 银脉（Silver Pulse）快速上手

> 给接手人 / 小爽：1 页看完"是什么、在哪、怎么跑、有哪些自动化、关键决策去哪查"。
> 细节与全量状态见 `TASKS.md`；权威方案与决策见 `产品方案与决策文档.md` 与 `决策记录_2026-07-09.md`。

## 1. 项目是什么（一句话）
一个帮小爽做"银发经济深度文章选题"的情报网站（GitHub Pages 公开）：自动抓海外银发新闻 → 打分排序 → 生成资讯看板与企业库，辅助"这周写哪个题、为什么写、国内有没有人写过"。

## 2. 目录结构（只列关键文件）
- `run_daily.py` — 每周全流水线编排（采集→打分→生成网页）
- `config.py` — 信源清单 / 评分维度 / 阈值（**别动 SOURCES 段**）
- `collector.py` `score_and_merge.py` `generator.py` `gen_enterprise.py` — 采集 / 打分 / 生成
- `selection/` — 评分、聚类、推荐理由、翻译等模块
- `data/` — 运行产物（`seen_urls.json` 去重、`scored_latest.json` 打分、`user_pref.json` 偏好）
- `index.html` `enterprise.html` `about.html` — 生成的前端页面
- `TASKS.md` `产品方案与决策文档.md` `决策记录_2026-07-09.md` — 任务 / 方案 / 决策记录

## 3. 跑一次完整流水线（3 步）
```bash
git pull                          # 1. 取最新
python run_daily.py               # 2. 跑全流水线，自动生成 index/enterprise/about.html
git add index.html enterprise.html data/seen_urls.json && git commit -m "daily run"  # 3. 提交产物
```
> 当前版本 ≈ 0 积分：翻译走免费接口，其余全代码；AI 增强（强模五维）暂关，需 key 才开。

## 4. 自动化列表
**每周日（WEEKLY）**
1. 资讯更新 11:00 — `run_daily` 全流水线
2. 标签池迭代 17:30 — 更新常现选题标签池
3. 摘要推送 19:00 — 本周精选摘要推给小爽
4. 自愈补跑 21:00 — 主更新失败当晚兜底重跑

**每小时（HOURLY · 兜底续命，错开）**
5. 整点续命（:00）— 读 `TASKS.md` 认领 🔴 任务并落地
6. 半点续命（:30）— 同上，与整点错开避免撞车（认领机制见 `data/autonomous_progress.log`）

## 6. 常见问题（FAQ）
1. **改了代码网页没变？** 必须 `python run_daily.py` 重新生成 html，改 .py 不会自动出页面。
2. **评分现在可信吗？** "信号强度 + 时效"最可信；旧行业/写作/反常识维度已整改（见 TASKS 评分冲突处理），强模五维待 key。
3. **会烧积分吗？** 当前流水线 ≈ 0 积分；只有开 AI 增强（0722 后）才计费。
4. **config 里两套评分定义冲突？** 以 `决策记录_2026-07-09.md` 三阶段为准，旧 `SCORING_DIMENSIONS` 已弃用。
5. **新增任务 / 改状态？** 直接改 `TASKS.md` 对应行 + `git commit`（main），并写 `data/autonomous_progress.log`。
