## 0. 先回答：标签到底有什么用（价值）

你问"除了分类、筛选、展示、搜索、企业关联，标签还有哪些价值"。我把标签的价值拆成 **8 个**，前 5 个是你说过的，后 3 个是我补的：

1. **分类补全（骨架 vs 正交）**——分类是"一企业一主业"的互斥骨架（13 个 L1），但一个企业常跨多个需求。标签是正交多维特征（一企业多标签），专门解决"跨赛道"表达。例：一家做认知症机器人的公司，分类只能挂"康复设备"，但标签可以同时挂 `认知症`+`养老科技`+`感官辅助`。
2. **选题雷达二次细分信号**——你说过评分要靠标签二次细分。标签是评分引擎的特征维度（比如按标签聚类算"赛道热度"，novelty 之外再加一层）。
3. **需求赛道聚合**——把分散在不同分类、但同需求的企业聚起来。例：`居家护理` 跨 养老服务/健康服务/医疗 三类，标签让它们在一个入口出现。
4. **搜索/SEO/关键词命中**——按"用户习惯的称呼"搜，而不是按我们的内部分类搜。
5. **企业关联/推荐**——相似标签的企业互为"以海外为镜"的候选（海外企业 ↔ 国内可比）。
6. **【补】反馈回灌的信号汇聚点**——你收藏/导出的反馈（`feedback.jsonl`）是按标签聚合的。哪些赛道被你反复收藏，就告诉评分引擎"这条赛道更值得推"。这是已经落地的 L3 回灌链路的输入。
7. **【补】聚类/去重依据**——同标签企业天然易聚成一簇，避免选题重复推（呼应现有 cluster 机制）。
8. **【补】跨源/跨期趋势监测**——标签层面的时间分布 = 赛道升/降温仪表盘（呼应 urgency/novelty）。比如某月 `养老科技` 标签企业集中融资，就是热点信号。

**结论**：标签不是"分类的补充说明"，它是**选题发现的主操作面**。所以分级（L1 筛选 / L2 搜索）比"全平铺"更利于你日常用。

## 1. 三层模型：分类 / 标签L1(筛选) / 标签L2(搜索)

```
分类(13 L1, 固定骨架, 一企业一个主业)   ← 不动
标签L1(一级标签, 筛选层, ~16个)         ← 显示在筛选栏，点一下即筛
标签L2(二级标签, 搜索层, 多多益善)      ← 全量进标签云/搜索框，可搜不可筛
```

- **L1（筛选层）**：数量少（你原话"一次展示太多了，不太好"）。入选标准 = **企业数够多** × **你/读者习惯这么叫**。每个 L1 也是可搜的。
- **L2（搜索层）**：更细、更多，作为关键词搜索用，"多多益善，还是要有价值，也别太多"。L2 挂在某个 L1 父级下（层级），点 L1 时自动带上它的 L2 子项。
- **筛选行为**：点 L1 `认知症` → 命中"带 `认知症` 标签，或带其父级为 `认知症` 的任意 L2"的企业。
- **搜索行为**：搜索框输任意词（含 L2，如"音乐疗法""助听器"）→ 全量匹配。

这样你日常：常用需求用 L1 点；冷门/专业词用搜索框捞 L2。两全。
