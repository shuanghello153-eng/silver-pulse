网站说明

Silver Pulse 银脉 · 银发经济投融资资讯聚合 + 企业数据库

资讯版定位

Silver Pulse 资讯看板是一个全球银发经济投融资信息聚合器，每天自动从海外和国内的核心信源采集银发经济相关的新闻资讯，为内容创作者提供选题发现服务。

我们的核心价值：以海外为镜，照中国之路。追踪海外银发经济企业的融资、收购、产品发布动态，为中国银发经济创业者提供路书。

评分机制

当前状态：评分功能已暂停
评分体系仍在优化中，看板暂时不展示评分。资讯采集和标签分类正常运行。

评分体系设计（暂停前的方案，后续恢复时参考）：

  • 行业相关度（35%）：与银发经济核心赛道的关联程度
  • 信号强度（30%）：融资金额、收购规模、参与方分量
  • 写作潜力（20%）：故事性、独特角度、中国可借鉴性
  • 时效紧迫度（15%）：新闻热度窗口，是否应本周处理

阈值设计：≥7.0 高价值 | 5.0-6.9 值得关注 | <5.0 归档

更新机制

  • 采集频率：每天 07:00 自动运行（自动化任务）
  • 采集范围：过去 7 天的资讯
  • 展示逻辑：精选条目优先展示，全量条目可切换查看
  • 排序方式：默认按日期降序（最新优先）

使用指南

  • 搜索：在搜索框输入企业名称（中英文均可）
  • 筛选：按地区（国内/海外）或分类（15类）筛选
  • 详情：点击任意行展开详细信息（简介、原始分类、融资、来源等）
  • 排序：默认按序号排列（先国内后海外，按入库顺序）

整体架构

Silver Pulse 银脉由三个核心模块组成：

  • 资讯看板（index.html）：每日银发经济投融资新闻聚合
  • 企业数据库（enterprise.html）：全球银发经济企业图谱
  • 网站说明（about.html）：本页面，规则与字段说明

更新频率

模块频率时间方式
资讯采集每日07:00自动化（RSS采集 + 标签分类）
企业库不定期—手动更新（新数据源融合时）
评分暂停中—规则优化后恢复

技术架构

  • 前端：纯静态 HTML + CSS + JavaScript，无后端依赖
  • 托管：GitHub Pages
  • 采集：Python + feedparser（RSS-first 策略，0个403错误）
  • 自动化：WorkBuddy 定时任务，每日 07:00 执行
  • 数据格式：JSON（结构化存储） → HTML（展示层生成）

数据流

资讯看板的数据流：

  • RSS 信源 → collector.py 采集 → raw_YYYYMMDD.json
  • 原始数据 → scorer.py 评分（暂停中） → scored_latest.json
  • 评分数据 + 原始数据 → generator.py → index.html

企业库的数据流：

  • 多源数据（书籍/PDF/xlsx/mapping）→ 提取脚本 → *_companies.json
  • 多源 JSON → merge_v2.py 融合去重 → all_enterprises.json（16字段统一schema）
  • 融合数据 → gen_enterprise.py → enterprise.html
