资讯版定位

Silver Pulse 资讯看板是一个全球银发经济投融资信息聚合器，每天自动从海外和国内的核心信源采集银发经济相关新闻资讯，为内容创作者提供选题发现服务。

核心价值：以海外为镜，照中国之路。追踪海外银发经济企业的融资、收购、产品发布动态，为中国银发经济创业者提供路书。

分类体系（事件类型L1 + 涉及领域L2）

每条资讯会被自动分类为一个事件类型（L1）和一个或多个涉及领域（L2）。分类与标签严格分开。

事件类型 L1（7 类）：

事件类型说明
融资企业获得新一轮融资
收购并购企业被收购或并购交易
政策法规政府政策、监管法规出台
产品发布新产品或新服务上线
行业趋势行业报告、数据研究、趋势分析
人事变动高管加入/离职、创始人动态
其他事件不属于以上类型的重要事件

涉及领域 L2（复用企业库13个一级分类名称）：

购物渠道, 日常用品, 健康食品, 老年文娱, 健康服务, 适老化改造, 行业服务, 养老地产, 养老服务, 养老用品, 康复设备, 失智老人赛道, 产业资本/投资机构

分类逻辑：事件类型通过关键词匹配自动判定（融资/收购/政策/产品/趋势/人事/其他）。
涉及领域通过领域关键词匹配，一篇文章可涉及多个领域。

标签池（TAG_POOL）

标签是与分类独立的自由标记，用于标注资本信号、背书、阶段、特殊模式等。每条资讯最多5个标签，所有标签来自预定义的标签池。

资本信号（4个）：

融资过亿 已被收购 IPO上市 已退市

背书信号（4个）：

AARP旗下 Khosla投资 YC孵化 政府支持

发展阶段（4个）：

种子轮 成长期 成熟期 已倒闭

特殊标记（6个）：

对标标的 模式创新 中国市场已有同类 订阅制 B2B2C 硬件+服务

地理标记（3个）：

日本 以色列 欧洲
标签池迭代规则：标签池每月迭代一次。每次迭代时，AI会扫描近一个月的资讯数据，识别高频出现但尚未纳入标签池的概念、赛道、模式，自动补充到TAG_POOL中。迭代后重新标注所有已入库资讯。
迭代成本：每月约消耗5-10K tokens（数据扫描 + 新标签识别 + 配置文件更新）。

数据筛选逻辑

为什么展示这些数据而不是其它？资讯看板的筛选策略分三步：

第一步：信源采集

  • T1信源：总是采集，尽量获取全文
  • T2信源：采集，获取摘要（summary）
  • T3信源：采集，仅用于相关性预过滤，不直接展示

第二步：相关性过滤

  • 文章标题+摘要必须匹配至少一个相关性关键词（RELEVANCE_KEYWORDS）
  • 匹配无关关键词（IRRELEVANT_KEYWORDS）的文章直接丢弃
  • 地区由信源决定，不由内容判断：国内信源→国内，海外信源→海外

相关性关键词示例（共78个，展示前20个）：

senior elderly aging age tech agetech older adult older adults aging population silver economy home care home health hospice assisted living long-term care long term care nursing home caregiver caregiving cognitive dementia

无关关键词（共10个）：

pediatric pediatrics children infant neonatal pregnancy maternity adolescent teen child

第三步：展示排序

  • 默认按日期降序（最新优先）
  • 精选视图：展示评分≥5.0或人工精选的条目（当前评分暂停，精选=全量，恢复评分后两者会不同）
  • 全量视图：展示所有通过相关性过滤的条目
  • 可按事件类型、涉及领域、标签、地区多级筛选
精选 vs 全量 说明：当前评分功能暂停中，所以精选和全量显示的数据完全一样。未来恢复评分后，精选只展示高价值条目（≥7.0）和值得关注条目（5.0-6.9），全量则展示所有通过过滤的条目。

推荐理由（AI分析）

资讯卡片底部的蓝色文字是AI对文章的分析点评（recommendation字段），不是原文摘要。内容包括：

  • 该资讯对银发经济领域的意义
  • 对中国市场的可借鉴之处
  • 选题建议或写作角度提示
积分消耗：推荐理由由AI生成，每条约消耗200-500 tokens。仅精选条目有推荐理由，全量条目不生成。
低成本方案：如果积分紧张，可关闭推荐理由生成，仅保留标题+摘要+分类。每次采集约节省3-6K tokens（62条×100 tokens/条均）。

评分机制

当前状态：评分功能已暂停
评分体系仍在优化中，看板暂时不展示评分。资讯采集和分类正常运行。

评分体系设计（暂停前的方案，后续恢复时参考）：

  • 行业相关度（35%）：与银发经济核心赛道的关联程度
  • 信号强度（30%）：融资金额、收购规模、参与方分量
  • 写作潜力（20%）：故事性、独特角度、中国可借鉴性
  • 时效紧迫度（15%）：新闻热度窗口，是否应本周处理

阈值设计：≥7.0 高价值 | 5.0-6.9 值得关注 | <5.0 归档

更新机制

  • 采集频率：每天 07:00（北京时间）自动运行
  • 采集窗口：过去 7 天的资讯
  • 每日积分消耗：约25-50K tokens（采集8-12K + 分类3-5K + 推荐理由12-31K + HTML生成2-3K）
  • 展示逻辑：精选条目优先展示，全量条目可切换查看
  • 排序方式：默认按日期降序（最新优先）
  • 标签池迭代：每月一次，AI扫描近月数据自动补充新标签

整体架构

Silver Pulse 银脉由三个核心模块组成：

  • 资讯看板（index.html）：每日银发经济投融资新闻聚合
  • 企业数据库（enterprise.html）：全球银发经济企业图谱
  • 网站说明（about.html）：本页面，规则与字段说明

语言规则

  • UI文字：全部中文
  • 企业名称：保留原始语言——中文名写中文，英文名保留英文不翻译
  • 信源名称：常见英文信源名称翻译为中文显示（如 Home Health Care News → 家庭健康护理新闻），保留原英文名在括号外
  • 数字：中文格式（如"5000万"而非"50 million"）
  • 日期：YYYY-MM-DD 格式
  • 标签：中文，2-6字，每条最多5个
  • 推荐理由：全部中文，2-3句言之有物，说明对中国市场的具体参照

分类与标签的区分

分类是结构化的层级体系，每条资讯/企业必须归属一个分类。
标签是自由标记，用于补充分类无法表达的维度（如资本信号、背书、阶段、模式特征）。

两者在UI上分开展示：分类用独立行展示，标签用蓝色badge展示，不混在同一行。
  • 资讯分类：事件类型L1（7类）+ 涉及领域L2（复用企业库13类名称）
  • 企业分类：一级分类L1（13类）+ 二级分类L2（70子类）
  • 标签池：资本信号 / 背书信号 / 发展阶段 / 特殊标记 / 地理标记

低成本方案（积分紧张时使用）：
1. 关闭推荐理由生成 → 节省12-31K tokens/次（最大节省项）
2. 减少精选条目数 → 从62条降至20-30条，推荐理由减少50%
3. 缩短采集窗口 → 从7天降至3天，减少采集量
4. 暂停T3信源 → 只保留T1+T2核心信源，减少采集量
5. 跳过分类关键词匹配 → 不分类，只做相关性过滤（不推荐，影响筛选体验）
推荐策略：优先关闭推荐理由生成（方案1），可节省约50%的每日积分消耗。

技术架构

  • 前端：纯静态 HTML + CSS + JavaScript，无后端依赖
  • 托管：GitHub Pages
  • 采集：Python + feedparser（RSS-first 策略，0个403错误）
  • 自动化：WorkBuddy 定时任务，每日 07:00 执行
  • 数据格式：JSON（结构化存储） → HTML（展示层生成）

数据流

资讯看板的数据流：

  • RSS 信源（L1域名→L2频道）→ collector.py 采集 → raw_YYYYMMDD.json
  • 原始数据 → 相关性过滤（RELEVANCE/IRRELEVANT关键词）→ 分类（事件类型+涉及领域）
  • 分类数据 → generator.py → index.html

企业库的数据流：

  • 多源数据（书籍/PDF/xlsx/mapping）→ 提取脚本 → *_companies.json
  • 多源 JSON → merge_v2.py 融合去重 → all_enterprises.json（16字段统一schema）
  • 融合数据 → gen_enterprise.py → enterprise.html
