> 状态：**仅方案，未写回任何数据、未部署**。等你拍板后再落地。

## 0. 先对齐：为什么要分三层（从本质出发）

一个问题可以有三种"切法"，粒度从粗到细，各司其职：

| 层 | 是什么 | 作用 | 是否互斥 | 谁来定 |
|---|---|---|---|---|
| **分类（L1/L2）** | 业务骨架 | 企业"主属哪行"——一级定大盘、二级定细分 | 每企业主属 1 个 L1 + 1 个 L2（互斥） | 已固定，不动 |
| **标签（L1/L2）** | 横切/更细的属性 | 补充分类没覆盖的维度（跨行共性、更细需求点） | 每企业可多标签（正交） | 本方案定 |
| **搜索关键词** | 同义词/别名 | 让搜索"听懂人话"，不进标签池 | 纯索引，不可见 | 规则派生 |

**为什么要标签？** 分类只能回答"它是干啥的"，但答不了：
- 它**面向谁**？（ToB / 女性健康 / 科技巨头下场）
- 它**用什么手段**？（远程医疗 / AI技术 / 数字化工具）
- 它处在**什么资本状态**？（融资/IPO）
- 它的**具体需求点**比分类更细的是什么？（分类只有"养老服务→临终关怀"，但你想细分到"殡葬服务/遗产规划"）

**为什么标签不能和分类重名？** 分类已经提供了该筛选维度，再建一个同名标签 = 重复造轮子、还让筛选栏变脏。上次我把"临终关怀"改名"临终与遗产"当成标签，本质是换皮不换骨——这是错的，本次彻底改。

**为什么要搜索关键词这一层？** 用户嘴里的词千变万化："助听器"≠"感官辅助"但应搜得到；"阿尔茨海默"应落到"认知健康"；"养老院"应落到"养老社区/养老地产"。这些同义词不该污染标签池，但搜索必须认。当前搜索**只匹配企业名**（见 §6 的 bug），这层几乎是空白，急需补。

## 5. 三层在页面上怎么落（筛选 / 列表 / 搜索）

| 层 | 页面位置 | 行为 |
|---|---|---|
| 分类 L1/L2 | 第2/3行筛选 | 点击展开 L2，主属过滤 |
| **标签（筛选子集）** | 第4行"标签"筛选栏 | 只放 §4.1 约 **20 个高频标签**，点击过滤（一次太多不好，你定的） |
| **标签（全量）** | 每张企业卡片下方 | 该企业所有标签都显示为小徽章（可点多可搜索） |
| **搜索关键词** | 顶部搜索框 | 同义词映射到标签/分类，输"助听器"也能命中"感官辅助" |

**关键点**：筛选栏只放"你最常用"的约 20 个；全量标签在企业卡片上展示（多多益善，有价值即可）；同义词只在搜索时生效，不污染标签池。

## 6. 搜索关键词层（含当前 bug 与修复）

**当前实现的问题**（`gen_enterprise.py:939`）：
```js
const searchMatch = !q || name.includes(q);   // 只搜企业名！
```
占位符写"名称/描述/标签"，实际**描述、标签、同义词都没搜**——是个假搜索。

**修复方案**：
1. 给每张卡片加 `data-search` 属性 = `名称 + 描述 + 标签 + 搜索同义词` 拼成的文本。
2. `filterEnt()` 改为 `data-search.includes(q)`。
3. 搜索同义词表（示例，规则派生，不进标签池）：

| 用户输入 | 命中 |
|---|---|
| 助听器 / 听觉障碍 | 感官辅助（助听） |
| 阿尔茨海默 / 失智 | 认知健康 / 失智老人赛道 |
| 养老院 / 敬老院 | 养老社区 / 养老地产 |
| 诊所 / 门诊 | 医疗 |
| 机器人 | 养老机器人 |
| 营养品 / 保健品 | 营养食品（注意区分 保健品&OTC 分类） |
| 退休 / 养老金 | 养老金融 |
| 保险 | 保险科技 |
| 智能家居 | 数字化工具 / 适老辅具 |
| 订阅 | 订阅制 |

## 7. 去重逻辑（海外以选题库为主，国内单独算）

> 数据事实：选题库 504 家，500 家已在系统（海外基本全收录）；系统"选题库未覆盖"825 家（国内为主）。

**规则**：
1. **海外企业**（在选题库里出现的）：标签直接用 **选题库 细分领域_1 → §4 映射**，不二次扫系统字段。避免 500 家被算两次。
2. **国内企业**（系统有、选题库无）：从 `category_l1/l2 + description + highlights + 现有保留标签` 推导，套 §4 同义词映射。
3. **绝不**把"选题库算一次 + 系统算一次"叠加到同一家企业。
