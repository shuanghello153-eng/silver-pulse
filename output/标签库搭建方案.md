## 二、标签库结构（12 个 L1 分组，命名可直接借用原分类）

用普通编号（1. 2. 3.…），避免歧义。L1 名尽量沿用你系统里好用的一级分类词。

1. **居家与社区照护** — 居家护理(←居家养老/家庭护理/上门医疗)、护理平台(←寻找护理/护理员支持)、社区养老、助浴助洁
2. **机构照护** — 养老社区、照护机构、专业级护理、养老机构SaaS
3. **医疗健康** — 医疗(←医护配置/中医)、慢病管理、康复、用药管理、健康监测、健康筛查、远程医疗
4. **认知与精神** — 认知症、痴呆症、精神健康、心理健康
5. **临终与遗产** — 临终关怀、安宁疗护、殡葬服务、遗产规划、遗嘱
6. **辅具与适老** — 感官辅助(助听/助听器)、适老辅具、适老化产品、防跌倒、肢体障碍、视力检测
7. **消费品** — 老年产品(←眼镜/假发/护肤/营养食品/食品营养)、保健品&OTC、长寿科技、鞋服(←自适应鞋/老年鞋)、服装、骨关节营养
8. **智能与科技** — AI科技(←具身智能/脑机接口/养老机器人)、数字化工具、智能硬件、IoT平台、语音AI、医疗AI
9. **文娱社交** — 文娱社交(←老年文娱/银发文娱)、社交平台、社交陪伴、老年教育、老年旅游、老人相亲
10. **女性健康** — 女性健康、更年期、女性盆底（更年期/盆底本就属女性，合并为"女性健康"）
11. **产业资本** — 融资/IPO(←有融资/融资过亿/已被收购/医疗REIT)、养老金融、产业资本
12. **行业服务** — 行业媒体(仅限真媒体)、咨询研究、运营商、行业展会、战略咨询

> 全量 284 个 L2 分类词必须逐一归入上述某组（映射全表见 `标签库现状审计.md`）。少数一次性词（头皮护理、骨胶原、头皮修护…）就近归入 7 消费品 等，绝不丢弃。

## 三、三套底座如何合并 + 同类词（关键修正）

底座：系统 47 + Excel 细分领域₁ 157 + L2 分类 284 ≈ 并集约 500–600 种。

**合并（取规范词，原词进同类词层）**：
- 居家养老 + 家庭护理 + 上门医疗 → **居家护理**；同类词：居家养老 / 家庭护理 / 上门医疗
- 寻找护理 + 护理员支持 → **护理平台**（注：护理平台是我新造的规范词，数据里无此原词；其同类词=寻找护理(Excel15)/护理员支持(5)/上门照护）
- 老年文娱 + 银发文娱 → **文娱社交**；同类词：老年文娱 / 银发文娱
- 银发消费 / 银发食品 → **消费品 / 食品营养**
- 医护配置 → **医疗**
- 助听 + 助听器 → **助听**（归 感官辅助）
- 资本类 → **融资/IPO**

**同类词层必须保留并展示**：每个标签的同类词在标签库里可见（如点开"护理平台"能看到"寻找护理、护理员支持"）。搜索后期接入同类词扩展。

**垃圾剔除**（不进标签库）：国家/年份/Crunchbase/APP/纯数字/对标标的/反共识/硬件+服务（语义太泛，拆进具体组）。

## 四、原分类不可信 → 从描述重打（本轮核心修正）

数据证实：全库 **344 家**被标 `行业媒体`，抽样 18 家只有 **2 家真媒体**，其余全是产品/服务公司。故：

- **绝不按 category_l2 直接转标签**。
- 方法：对每家企业读 `name` + `description`，用关键词映射到 L1/L2；category_l2 仅作**提示**。
