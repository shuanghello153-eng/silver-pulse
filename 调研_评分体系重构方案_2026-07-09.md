## 二、信号强度公式（已编码于 selection/signal_strength.py）

| 分量 | 规则 | 满分 |
|---|---|---|
| 信源起点 | T1=1.5 / T2=1.0 / T3=0.5 | 1.5 |
| IPO/上市 | 命中即 +6（巨头，招股书/年报信息量丰富） | 6 |
| 融资金额(海外$) | ≥10亿=5 / 1亿=4 / 5000万=3 / 1000万=2 / 500万=1 / 100万=0.5 | 5 |
| 融资金额(国内¥) | ≥10亿=6 / 1亿=5 / 5000万=4 / 1000万=3 / 500万=2 / 100万=1 / 50万=0.5 | 6 |
| 收购金额($/¥) | ≥100亿=6 / 50亿=5 / 10亿=4 / 1亿=3 / 5000万=2 / 1000万=1 | 6 |
| 时效(动态) | 近1周=2.5 / 1月=2.0 / 2月=1.5 / 半年=1.0 / 1年=0.5 / 更久=0.25 | 2.5 |

- **时效动态计算**：存发布日期，每次评分时实时算"距今天数"→ 自然衰减，无需每月重算任务。
- **国内投融资物以稀为贵**：国内分级上限 6 分 > 海外 5 分（国内现在融资极少，单笔更值钱）。
- 自测：C轮2亿$ = 7.5 / 80亿$收购 = 8.0 / 纳斯达克IPO = 9.0 / 国内3000万天使 = 5.0 / 老年游戏 = 3.5。

## 三、企业 + 资讯 统一一套方案（建议采纳）

你提的"企业和资讯就用一套方案"我赞同。现状是资讯有 5 维、企业有 research_value，两套口径。统一方向：

- **共用信号强度**：企业的最新动态（融资/并购/IPO）直接复用阶段3公式 → 企业库"近期动态"按同一信号强度排序。
- **共用强模型五维**：资讯跑信息量/差异化/可复制时，顺带把关联企业更新进企业库（抓取最新信息+关联动态）。
- **可复制维度直接调用企业库**：判断"国内有无同类赛道/有无融资上亿或上市公司/支付方差异（美 Medicare/自费 vs 中 医保/自费 vs 日 近似中国）"——这正是企业库已存的字段，零额外搜索即可判定。
- **分级统一**：精选阈值一套（如 high≥7 / watch≥5），资讯和企业共标。

> 注意：当前代码是两套管线，统一需要重构 generator/enterprise_score 的调用链。建议等评分体系整体拍板后，挂到"周日常跑"里一次性切换，避免半吊子。
